I ran the majority of my home lab and smart home from a single network-attached storage (NAS) enclosure for years, relying on first-party apps for much of the heavy lifting. If I were to travel back in time and inform the past version of Richard that he'd fully make the switch to Proxmox and build out multiple nodes to handle everything, I would have responded with "That seems a little excessive for what I need," but requirements have most certainly evolved over time.
✕

For those golden NAS-only years, 
Synology’s ecosystem
 felt like the perfect balance of simplicity and power with a polished GUI atop one of the best prebuilt OS options around. The brand was great for creating something with reliable storage and offered just enough container support to run everything I needed. Docker on Synology gave me media servers, automation tools, and monitoring stacks. It was mostly hands-off, which was neat. I didn't feel like I had to log in too often.
But as my home lab started to evolve outside of the NAS, things needed to change. I made the jump to bare metal solutions, but this was more hassle than it was worth. Enter Proxmox, which was extensively covered by my esteemed colleague Ayush Pande. Once I tried it out, I was immediately hooked. Making the switch from NAS to Proxmox nodes didn't just improve my hardware and software stack, but the hypervisor fundamentally changed how I approach technology, particularly running services and trying out software.

                        Synology gets a lot of flak
               

            But it's still a solid prebuilt NAS brand
    
I know there are many out there who aren't too pleased with Synology's more recent moves as a company, 
restricting hard drive support on newer models
, to name but one example. That all said and done, Synology still remains as one of the most user-friendly options for beginners to pick up and use their first NAS, unlocking access to the self-hosting world, saving money, and learning something new. If you find TrueNAS and Unraid utterly terrifying, Synology is the way to go.
✕
 Remove Ads
First-party app support is generally fairly solid, and you won't have to touch Docker unless looking to self-host some of the more obscure containers and software. But when you do eventually dip your toes into community-provided packages, things are still fairly grounded thanks to Docker's intuitive UI within Synology DSM, and options like Portainer make it almost painless. There's still no reason to open up a terminal window and learn a new language in a few commands.
But Synology can be limited, just like other NAS units. These enclosures are fantastic in that they offer some serious firepower for handling tasks other than storing files, but you'll still eventually encounter resource contention, networking feels a little clunky between containers, and mapping persistent storage can feel messier than it should be. General debugging an prove challenging too, especially when mixing first-party apps and Docker containers. That's where Proxmox truly shines.
This hypervisor platform is completely open. You take what the system has with the internal components, CPU, RAM, and storage, and share it among all the configured virtual instances. This could be a virtual machine running Windows or a Jellyfin Linux container to stream all your media.

                        Making the switch to Proxmox
               

            Moving almost everything across
    
✕
 Remove Ads
When looking into     
Proxmox
, I initially assumed I'd have to have a separate virtual machine (VM) running per service. It's what usually happens with multiple OS environments, each with dedicated resources, but it couldn't have been further from the truth. Enter Proxmox Linux containers (LXC). This is where everything suddenly clicked, and the magic started to happen. Unlike running Docker and containers within a VM, Proxmox allows for LXCs to run as more lightweight instances.    
Each has its own filesystem, users, and dedicated package manager. It's more like running a barebones Linux installation than launching a container within Docker. Using LXCs unlocked a more responsive platform for multiple services running on the same node. And because I was able to pick up some cheap compact systems, I've migrated all my home lab to the final stage: a multi-node cluster with redundancy and shared pools. Using low-powered hardware ensures we keep bills low without sacrificing performance.
And that's what makes LXCs almost a requirement for this setup. Compared to DSM, which had Docker run on top of the OS, Proxmox cancels out some of the layers, allowing services to run closer to the host kernel. It may not sound like much, but this reduces overheads and improves performance. It makes assigning resources per instance easier since everything is more predictable. My dashboard loads quicker, Jellyfin doesn't take as long to index, and the entire home lab feels more responsive.
✕
 Remove Ads
That's not to say Synology NAS or custom installations with some free OS isn't the right choice or offers subpar experiences. I simply find Proxmox to be more efficiently utilize available hardware. Unlike some other solutions, Proxmox clearly shows precisely how many resources are being used by each instance. It's all clear as day in graphs on each node, and I have full access to all machines within the single web interface. It's like running my own mini data center, and it's brilliant.

            Proxmox made my home lab easier to manage
    
One sung hero of Proxmox is how well it fits into a backup strategy. Before, I'd have to back up volumes and pray that compose files would continue working after recreating containers, and troubleshoot any potential mismatches. For Proxmox, I simply create a backup for each specific container. If something breaks, I restore the entire backup, which sorts everything out, including the file system, configuration, and packages.

Isolation is favored within Proxmox, which was a different approach to what I was previously exposed to. Docker would lead you to cram as much into a single stack as possible for less work, but Proxmox forced me to isolate everything. A container monitors the stack. Another handles media management, and I even have databases separated for specific containers. It all just works and is simple to document and keep tabs on, even as the home lab continues to grow.
✕
 Remove Ads
Even with this refined setup, keeping a Synology NAS is still worthwhile to store data and offer volumes and pools to the network. This can easily be integrated into Proxmox, providing remote storage for various containers. My NAS became a place to store data and Proxmox handled running the home lab.
