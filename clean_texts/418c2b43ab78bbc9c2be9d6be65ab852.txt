VS Code is probably the 
best IDE out there
 right now. It has almost everything you need, whether it’s AI features, stability, or performance. In terms of pure features, it checks all the boxes, and on paper, it’s open source. There isn’t much to complain about at first glance.
✕

That said, VS Code is not fully open source. The telemetry Microsoft collects is still a concern for some people, and the marketplace itself isn’t truly open. Publishers have to agree to Microsoft’s licensing terms, and the marketplace APIs aren’t available for use outside their ecosystem.
This becomes a real limitation if someone wants to 
fork VS Code
. They can’t legally use the official marketplace, which makes it harder for alternative IDEs to grow. That’s one of the biggest blockers for a more diverse ecosystem.
For example, 
VSCodium is a de-Microsofted version of VS Code
 that I recently switched to. It doesn’t have access to the official marketplace; instead, it uses the Open VSX Registry. Open VSX is open source and acts as a solid alternative to the VS Code Marketplace. In my experience so far, it’s actually been better.

                        Open VSX checks all the boxes
               

            It's designed to support editors built on the Visual Studio Code extension model
    
Open VSX Registry is an open-source extension registry designed to support editors built on the VS Code extension model. It is maintained by the Eclipse Foundation and provides a vendor-neutral alternative to Microsoft’s extension marketplace.
✕
 Remove Ads
Open VSX supports the same extension format used by Visual Studio Code. Extensions published for VS Code can be packaged and distributed through Open VSX without requiring structural changes. This ensures good compatibility across editors that implement the VS Code extension API, including VSCodium and other VS Code-based IDEs.
The registry provides standard marketplace features, including extension discovery, installation, versioning, and updates. You can browse available extensions, install them directly within supported editors, and receive updates through the same extension management interface. For most commonly used extensions, such as language support, themes, linters, and productivity tools, you will find equivalent versions easily available on Open VSX.
Open VSX is fully open source, including both the server-side registry and associated tooling. The source code is publicly available, and this transparency extends to the publishing process as well. Extension authors can independently publish their extensions without being restricted by a single vendor-controlled ecosystem.
The platform also supports self-hosting. You can deploy your own Open VSX registry instance to manage and distribute extensions in controlled environments. Although it’s overkill for an individual, it is especially useful for large organizations that need to meet specific compliance requirements. Self-hosting also comes in handy when you require offline access or internal curation of extensions.

                        OpenVSX is truly open source
               
✕

            And has almost every extension you need, with a few exceptions
    
Unlike Microsoft’s marketplace, Open VSX operates under an open governance model. It is not tied to a single commercial product or distribution. Extensions hosted on Open VSX can be used across multiple compatible editors without licensing restrictions tied to a specific vendor’s build or service.
The registry also supports automated publishing workflows. Extension developers can integrate Open VSX publishing into their CI/CD pipelines, enabling consistent updates across both Microsoft’s marketplace and Open VSX, if desired. This also allows devs to support a broader user base without maintaining separate codebases.

So it goes without saying that OpenVSX is not without its limitations. For one, it's up to developers to publish their extensions separately. This means some extensions available on the VS Code Marketplace are missing or don’t receive updates at the same time. This is particularly true for extensions published exclusively by Microsoft or those with restrictive licensing terms.
Still, Open VSX is not a bare-bones fallback. Its public catalog already includes extensions for the day-to-day tools most developers rely on, including Python, Java, PHP, Ruby, Swift, SQL, OpenTofu, and AI-assisted coding. Popular extensions such as Pyrefly, Language Support for Java by Red Hat, PHP, Composer, PHP Profiler, and Ruby LSP are available, and I haven’t had any issues using them.
✕

The Python extension brings support for IntelliSense, debugging, linting, formatting, refactoring, and unit tests. The Java, PHP, Ruby, Swift, OpenTofu, and SQLTools extensions cover full language support, debugging, database work, infrastructure tooling, and other common development workflows.
It's worth pointing out that Open VSX does not need to match Microsoft’s marketplace extension for the extension to be useful. It needs to cover the extensions you reach for every day, and in most common setups, it does exactly that.
