Self-hosting demands constant attention and maintenance. It sounds easy because most of us think that we need to just buy a mini PC, a bunch of Raspberry Pis, or use an old laptop, and the road ahead is easy. However, not all self-hosted tools are simple to run and manage. Some need continuous tweaking, adjustments, and monitoring to get the desired experience.
✕
 Remove Ads
In my self-hosting journey, I've used a lot of media streaming servers, 
local cloud storage
 solutions, remote file access and download servers, toolkits, and monitoring software. However, a few of them have never been replaced and work flawlessly without any daily efforts. Let's explore some of these.

                        Vaultwarden
               

            A competent password manager
    
Vaultwarden is an underrated gem that I host on a Raspberry Pi Zero 2W. It's the most lightweight implementation of a password manager that I host in my home. At the core, I use 
DietPi
, which makes the Vaultwarden warden install a one-click task.
Thereafter, this self-hosted password manager doesn't need anything more than a one-time password import and setup. Last month, 
I migrated from LastPass to Vaultwarden in one night
, and it's been smooth sailing since day one. I use the browser extension to access my passwords on the local network and often use the random password generator present in the toolkit.
You'll be surprised when comparing the features with a paid password manager service. I do complete system backups of DietPi to a USB drive as a fail-safe method because data redundancy is crucial when you locally host something sensitive like passwords.
✕

                        Tailscale
               

            Remote access and security
    
 It's the silent network protector of my home server that runs on each device and lets me connect to them when I'm outside. Home networks make accessing my self-hosted services extremely easy, but if I want to do so on someone else's Wi-Fi or mobile data, Tailscale helps me with it.
Setting up Tailscale
 hardly takes a few minutes, as I only need to install the app or package on the device and add it to the network. Thereafter, I only need to replace the IP of the server device with the Tailscale IP and specify the appropriate port number to view the web interface.
Most services play nicely with Tailscale, including Jellyfin and NextCloud. I've not had success with Plex while trying to do remote playback using Tailscale. It boots up with the OS, consumes unnoticeable memory in the background, and never falters.

                        Uptime Kuma and Dockge
               

            Docker monitoring and deployment
    
Uptime Kuma
 is a container monitoring tool and does a fantastic job in my workflow. Ever since its deployment, I've added monitoring items for all of my containers individually. It pings the services frequently and reports the status in a pictorial format.
✕
 Remove Ads
Since it has no control over the containers and can only do tracking and push notifications, you don't need to do much after the initial setup.
The same developer is the brains behind Dockge, which lets me create stacks using compose files. I don't like using the terminal for Docker compose files, and 
Dockge offers a simple interface to define the stack items and environment variables
. It can control container activity, but has limited control if you create containers using other tools like Portainer.

                        OmniTools
               

            A bunch of useful tools
    
 I've been a fan of 
OmniTools
 from the day I found that it could do a lot of tasks that would otherwise need a web tool. Since it is self-hosted, I don't have to upload any files to a server I don't know about. OmniTools works flawlessly as a Docker container and can easily convert my files into different formats, perform PDF edits and video file conversions, or do compression tasks.
I rarely need to worry about updating it or fiddling with settings after its deployment. The tool doesn't need any permissions, stores everything locally, and saves me the effort of visiting web tools or buying Adobe Acrobat for minor edits. I also used Stirling PDF in the past, but OmniTools has enough features for my use case and is extremely conservative about system resources.
✕
 Remove Ads
So, you can use a lower-end SBC to host it instead of deploying it on a mini PC or a powerful SBC like a Raspberry Pi 5.

                        Pi-hole
               

            Personal DNS resolver and ad blocker
    
Credit: 
 The invisible shield of my home network is Pi-hole, and it works 24/7 in the background without hiccups. Pi-hole is one of the most deployed DNS resolvers with built-in ad blocking and tracking prevention features. I evade my invasive ISP tracking and route all my requests via the Pi-hole server.

Pi-hole acts as an intermediary between what websites I want to open and the ads and tracking implemented on them. I can even switch 
Pi-hole
 lists that contain the details of known trackers. It's one of those services that a Raspberry Pi Zero 2W can handle with ease, but the lack of a dedicated Ethernet port makes it a bit problematic.
Still, I don't need to use anything while browsing the web or opening apps, as Pi-hole takes care of everything. It doesn't allow any ads to load or trackers to snoop on me, but it can sometimes break connections. The idea is to switch to updated lists and add a backup DNS resolver for your devices. However, the elaborate dashboard, lightweight memory profile, and simple setup are the reasons I prefer it over AdGuard.

            Not all self-hosted services demand attention
    
I'm a fan of Pi-hole and Uptime Kuma, as both work incredibly well after the first deployment. Vaultwarden is another gem that I never had to worry about after I began using it instead of a paid password manager.
Resource-heavy tools like NextCloud and Paperless-ngx need frequent tweaks to work correctly. I use them in combination with Immich for photos, but I cannot call them “deploy once and forget” tools.

Tailscale only uses a third-party server for configuration and UDP hole punching. The actual VPN connections are peer-to-peer - Tailscale used Wireguard for that. You can self-host Headscale if you want to host a coordination server yourself rather than using Tailscale's one. 
You can get a similar mesh network with Wireguard, but you need to manually configure it - every node needs to have every other node listed in its config. Tailscale does that for you automatically, plus it uses UDP hole punching so you don't need to open ports, plus each device on your Tailnet (Tailscale network) automatically gets a subdomain so you don't need to memorize IP addresses, plus the Tailscale IPs work both on your LAN (where it'll connect directly to the other device via the LAN) and over the internet, avoiding the need to use split horizon DNS (just use the Tailscale IP everywhere), plus you get ACLs which let you share a VPN node with a friend but only allow them to access particular ports.
Plenty of advantages of Tailscale, and the client code is all open source. The server is the only part that's closed-source, but like I mentioned, you can self-host Headscale which is an alternative implementation.
