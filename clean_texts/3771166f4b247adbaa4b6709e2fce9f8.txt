You don’t have to get bogged down by boring, repetitive dev tasks that you can set and forget.
At least, that’s the promise of the newly introduced Claude Code Routines. I was quick to test it, and it delivered.

If you’ve been using Claude Code for a while, you already know it’s great for automating the 
software development cycle.
But the catch has always been automation still needs you to manage the cron jobs, infrastructure, and MCP servers.
Claude Code Routines is designed to fix this problem.
You define an 
automation once
:
 a
prompt, the repository, and your connectors, then tell Claude when to run it on a schedule, via an API call, or in response to a GitHub event.
Like 
Claude Managed Agents
, this also runs on Anthropic-managed cloud infrastructure, so it keeps running, whether your terminal is on or off.

What Are Claude Code Routines?
A routine is a saved Claude Code configuration that includes a prompt, a GitHub repository, and your connectors packaged once and scheduled to run.
Unlike other automations we have set up before, routines execute on Anthropic-managed cloud infrastructure.
What makes it different is that Claude runs the task, and you see the results next time you check in.
