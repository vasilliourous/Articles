                        Caddy can be incredibly powerful
               

            Modules are the spice of life
    
Caddy was built as a web server
, but the modular construction makes it great for a whole variety of specialized tasks, like setting up an encrypted reverse proxy. Exposing multiple self-hosted services can turn into a complex management game, but it doesn't have to be that way. Caddy's single-file management method makes it easier, as it takes a few minutes to set up, and you can easily add to it as your home lab grows.
And if you find you need a new feature, Caddy lets you rebuild the image easily with the new module(s), so you can have the reverse proxy of your dreams. I don't have a ton of modules activated on mine, partly because I keep changing things, but the one module I never go without is the Cloudflare module so I can use my own domain, and handle DNS challenges for Let's Encrypt certificates for every service I run.

                        Building custom Caddy images is simple
               

            And you can do it in multiple ways
    
To use modules in Caddy, you have to rebuild the image to add the new features. The good news is that there are a ton of features you can add, but it's also complicated because you don't want to add too many modules and end up with something hard to manage. You can always rebuild the image whenever you want to add more.
For example, adding these non-standard modules by building a new Caddy image with xcaddy would look like:
xcaddy build \
 --with 
github.com/caddy-dns/cloudflare
 \
 --with 
github.com/mholt/caddy-dynamicdns
 \
 --with 
github.com/corazawaf/coraza-caddy/v2
 \
 --with 
github.com/caddyserver/cache-handler
 \
 --with 
github.com/ueffel/caddy-brotli
Then, once you've got the list of modules you like, bake it into a Dockerfile like so, then call that in your docker-compose file by using "
build: ."
 instead of 
"image:"
.
FROM caddy:builder AS builder
RUN caddy-builder \
github.com/caddy-dns/cloudflare
github.com/mholt/caddy-dynamicdns
 \
FROM caddy:latest
COPY --from=builder /usr/bin/caddy /usr/bin/caddy
Caddy also has a 
web configurator
 where you can select the modules you need, and download a fully built package for use on your machine. It's much easier to do things this way, especially when you're just starting out learning how Caddy works. Copy the URL from the Download button, and use 
sudo curl -o /usr/bin/caddy [url]
 to get it downloaded, then make it executable with 
sudo chmod 755 /usr/bin/caddy
.

Once you've got your Caddyfile, it's time to fill things like your Cloudflare API, so you can use it for DNS challenges and get HTTPS to your self-hosted services:
export CLOUDFLARE_API_TOKEN="your-api-token"
caddy run --config /etc/caddy/Caddyfile --adapter
