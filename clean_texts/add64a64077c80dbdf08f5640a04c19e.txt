                        Webtop
               

            A full desktop in your browser
    
Running a desktop environment inside a container sounds wrong until you try it. Webtop is exactly that: a full-fledged Linux desktop in a browser, able to be spun up in seconds. The practical upside is simple, you get a disposable Linux desktop you can reach from any device on your network, without spinning up a VM, configuring GPU passthrough, or maintaining yet another full OS install.
In real use, this becomes an isolated workspace for tasks you’d rather isolate: testing sketchy downloads, logging into secondary accounts, troubleshooting a web app in a clean environment, or running a tool that’s annoying to install locally. It also shines in a home lab because you can host it on a box tucked away in a closet, then pull it up from your main PC, a laptop, or even a tablet.

                        Personal AI assistants
               

            It never dawned on me that Ollama could run in Docker
    
✕
 Remove Ads
I recently set up a local LLM with 
Ollama
 in a dedicated Proxmox VM, but I had no idea I could deploy it using Docker. “Local AI” is usually framed as a hardware flex: big GPU, lots of VRAM, and a stack of models you benchmark for fun. But the more interesting use case is boring in the best way. A personal assistant you can open in a browser, ask questions, summarize text, draft messages, and iterate on ideas, without shipping everything you paste into it to a third-party service.
In practice, Docker makes this approachable because you can treat the whole AI setup like an appliance. One container handles the model runtime, another provides a clean web interface, and your persistent storage keeps models and settings intact across updates. If you don’t like the results, you can tear it down and try a different setup without leaving a mess behind. If I were to try that with my current setup, it'd be a bit more difficult than if I had just used Docker from the start.

                        Teamspeak server
               

            Not a full Discord replacement, but good enough for simple chat
    
With the recent changes to Discord, some of my online friend groups have been weighing the options of moving to another platform. Teamspeak isn't a Discord replacement in a 1:1 way, and I'm not pretending it is, but it's incredibly easy to spin up in a Docker container.
✕
 Remove Ads
TeamSpeak in Docker feels different because it’s self-contained. You keep your data persistent, you expose the ports you need, and the container becomes a replaceable wrapper around a stable service. If you move hosts, you migrate the volume and you’re basically done. If you want to roll back, you can do that cleanly too. Even if you’re only running it for a handful of friends like I am, Docker makes the whole thing feel less like “running a server” and more like “turning on a service."

                        A Samba file server
               

            Instantly turn anything into a NAS
    
Getting a shared folder set up can quickly spiral into a dedicated NAS OS, a bunch of plugins and a UI you don't really need. Running Samba in Docker is precisely the opposite. It’s a straightforward way to expose storage on your network using the same file-sharing protocol most people already rely on. It’s especially handy if you already have a machine running other services, and you want to add basic file sharing without turning the whole thing into a NAS.

                        Game servers
               

            Portable, easy to clean up
    
Yes, Minecraft is the most common example, but the pattern applies to a ton of games. A server is often just an application with a few ports, some configuration, and a persistent directory for maps or other save data. Docker makes it simpler, and what actually changes in practice is how safe it feels to experiment. Want to test a new mod pack, tweak settings, or spin up a second server temporarily? You can do it without risking your “real” setup.
