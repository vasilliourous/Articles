Most proxy servers, sometimes called 
forward proxies,
 can be used to obscure your location from whatever service you're connected to. Reverse proxies do almost the exact opposite—they sit between your self-hosted services and whoever is trying to connect. It might not sound that exciting, but it is one of the most useful things you can run for your homelab.

                        Self-hosting gets tedious
               

            Who wants to type IP addresses all day?
    
When you first start self-hosting, you're going to spend a lot of time typing in IP addresses to access your local services. Most of the time you only have to memorize two or three numbers—whatever is at the end of 192.168.0.
XXX
 or 10.0.0.
XXX
, but once you have a handful of services, it gets old fast. Did the Minecraft server end in 222? Which port did the WebUI for Proxmox need?
Luckily, there is a convenient solution: a reverse proxy. With a reverse proxy, you can assign normal, easy-to-remember names to your services instead of memorizing strings of numbers.

minecraft.home
 instead of 192.168.0.111:25565. If I were to open up a Jellyfin server, I could just assign it the name 
jellyfin.home

There is a little bit more setup up front, but if I'm going to have to type an IP more than 10 times, that little bit of extra work is well worth the trouble.

                        You don't need to expose your services to the web
               

            Reverse proxies can be local-only
    
Credit: Hannah Stryker / How-To Geek
✕
 Remove Ads
 Reverse proxies usually come up when you're looking into directly exposing your services to the web, but that isn't strictly necessary. I host a reverse proxy just for convenience.
Because nothing is exposed to the outside world, I don't have to contend with the hurdles that come with 
ensuring that everything is as secure as possible.
Security through obscurity only gets you so far. Never depend on "flying under the radar" to keep your services safe if you do expose them to the internet.

            HTTPs runs locally
    
Normally, if you type in a service's local URL or IP address, your browser is going to 
give you a safety warning
. It's a little annoying when you're self-hosting, but it is invaluable when you're surfing the internet, since it lets you know that you've connected to a website that 
isn't 
using the appropriate encryption to secure your data.
That is another small advantage of a reverse proxy. You can set up HTTPs for your services so you don't get the annoying warnings about an insecure page.

                        Which reverse proxy should you use?
               

            Caddy is easy to pick up
    
Caddy
 is one of the most popular reverse proxies for homelabbers due to its simplicity. You configure Caddy by modifying a single file—tell it the IP of the service, assign the service a name, and you're done.
It also sets up HTTPS automatically, though you can tweak its behavior as needed.
Caddy's documentation is also excellent, especially if you've never dug into anything related to DNS servers, HTTPS, or reverse proxies before. It is my first recommendation for anyone trying it for the first time.
✕

            Nginx Proxy Manager (NPM) has a graphical user interface
    
Nginx Proxy Manager (NPM)
 is another popular choice among homelabbers. There are three major points that distinguish it from Caddy:

                                        NPM has a user interface
                        

                                        NPM requires more manual configuration, especially if you want 
HTTPS

                                        NPM is primarily deployed using Docker
                        
However, besides those differences, it is just as capable as Caddy. If you like a user interface, or if you're looking for something that offers you more granular options right out of the box, NPM is probably for you.
If you're already running a lot of Docker containers, NPM is also a very natural option, since that dramatically simplifies the installation process.

                        Getting your services on the web will be much easier
               

            Your future self may thank you
    
Credit: Patrick Campanale / How-To Geek
 Eventually, you're probably going to need to put one service or another on the internet. In my case, they're usually game servers. Setting up a reverse proxy now makes it much easier to get a service on the internet in the future.

port forwarding the appropriate port
 on your router.

When you're done, you just remove the port forwarding rule and your network goes back to a secure local-only setup.

            A reverse proxy is a massive quality of life upgrade
    
A reverse proxy isn't a flashy feature. It doesn't really 
add 
anything exciting to your homelab. However, it does make everything else you use more easily accessible.
You get to stop memorizing IP addresses and ports, and if you configure HTTPS, you won't get pestered about the security of the service you're hosting.
It'll take an hour or two to set up if you've never done it before, but it'll be one of the best improvements you can make to your homelab.
