Recently I decided to get back into programming. I'm not following the convention of getting started in Visual Studio Code, though. Instead, I'm starting from scratch with the Kate advanced text editor.
While starting out in Rust programming, I was reading several guides that encouraged me to use VS Code as my starting point. It's free, and support for just about any language is there already or readily accessible with a promoted extension down. I'm learning on a different platform, though.

                        I'm already steeped in the KDE ecosystem
               

            Kate comes from the same family as Kubuntu and Plasma, which I use already
    
 I use Linux as my primary desktop, and I have for years. My favorite desktop environment to work in is KDE Plasma. Kate is developed under the same roof, so to speak, at the KDE Project.
In fact, Kate is sometimes packaged with KDE Plasma desktop environments. It integrates the Konsole terminal emulator, which I already use for command line work, and its theming options match the standard KDE Plasma themes. A single cohesive ecosystem makes me feel at home and in tune with my PC, and since it's KDE-based, I don't have Microsoft looking over my shoulder.

                        Kate is fully open source
               

            Believe it or not, VS Code is proprietary
    
Credit: Lucas Gouveia/How-To Geek
 Whenever possible, I prefer using open source software in place of proprietary options. So long as they're well-maintained and have the features I need, I opt for the solutions that keep me free of using a black box whose function is unknowable to me and anyone else who doesn't work for the corporation or entity behind it.
✕
 Remove Ads
While you can find 
VS Code's essential code base on GitHub
, going by the name "Code - OSS," the software packaged as VS Code actually comes with "Microsoft-specific customizations." In other words, the app you're using is not purely open source. In contrast, the final product known as Kate is completely open source.
Not to mention, Kate is more lightweight than VS Code, consuming about half the resources that VS Code does on my device. That means more portability; I can use Kate on lower-resource computers with less overhead.

                        I'm not interested in vibe coding
               

            Dear Copilot: Thanks, but no thanks
    
Credit: Lucas Gouveia/How-To Geek | SAG stock/
Shutterstock
 When I open VS Code these days, I'm greeted with multiple prompts to start prompting—prompting an LLM, that is. VS Code is really determined to set me on the 
vibe coding
 track with its Copilot integrations.
All of those features taking up space in the interface just don't appeal to me. If I chose to vibe code, then I might end up with seemingly usable software sooner, but I won't know how it works. I won't be able to troubleshoot it except to prompt an LLM to fix it, which it may or may not do—I'd have no way of knowing.

No, I prefer to understand the inner workings of software I'm attaching my name to. In short, I want to have an actual marketable skill, and that's not what "prompt engineer" is. That's to say nothing about the ethical and legal issues surrounding LLM use. I'd prefer to just keep my hands clean, and I don't want those dirty parts cluttering up my text editor.

                        I want to learn as much as possible
               

            I'd rather become an expert than a good-enough amateur
    
Credit: Lucas Gouveia/How-To Geek | Prostock-studio/
Shutterstock
 Speaking of learning, I appreciate the fact that Kate doesn't do as much hand-holding as VS Code does. Not only is there no LLM ready to do my homework for me, but there's no automatic prompt install extensions that give you huge assists in learning code.
For example, I had to install packages like 
rust-analyzer
, 
pylsp
, and 
marksman
 in order to get detailed support and linting for my code when writing in Rust, Python, and Markdown, respectively. This took some research to figure out, but once I did that, I had a deeper understanding of how text editors and IDEs work than I would have 
setting up VS Code for development
.
When I think about why I'm programming, it's not just for the sake of getting a product made. Sure, I'll be happy when I have software I can show to people and put to good use. It's important to me though that I actually develop skills and deepen my understanding of the subject matter as I learn.
✕

            I'm using it for more than just coding
    
I'm actually learning to not write only Rust in Kate. I'm also using it for Bash scripting and for Markdown writing. In fact, I made my first draft of this article in Kate. I also recently wrote 
a Bash script for cleaning my Downloads folder
, and that I composed entirely in Kate.
