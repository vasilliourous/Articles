Running a 
home lab
 starts simply with a few containers for media and photo backups, and maybe Home Assistant to build out a smart home.
 But once you start going down the rabbit hole, the setup grows fairly quickly. More services get added in over time, more devices, and all too soon, if you're like me, you're managing a small infrastructure instead of just a simple hobby project. Trust me, when you're that deep into the hobby, visibility starts becoming a real problem.
✕
 Remove Ads
Sure, a container manager like Portainer will show you containers and your network setup. Unifi, in my case, will show network clients, but what you really need is a unified dashboard. While each tool does its own job well enough, none of them gives you a complete picture of how everything connects. Predictably, when something breaks, you need to 
open 
multiple dashboards
 just to begin
 diagnosing the issues.
To solve that problem, I turned towards the open-source HomeLabInfo. This self-hosted project is designed for home lab users who want one place to track both their Docker containers and network devices. Instead of treating containers and infrastructure as separate things, it pulls them into one single dashboard so you can get an overview of your entire system as a whole and handles everything from network discovery to Docker monitoring, device mapping, and topology visualization.

                        HomeLabInfo makes container visibility much easier
               

            One dashboard for Docker health, system stats, and more
    
Credit: 
 Step one of using HomeLabInfo is the installation, which is simple enough via 
Docker
. The developer has even provided a compose file on the GitHub page. HomeLabInfo taps into host networking to properly scan your actual LAN, rather than only seeing Docker's isolated internal network, which is critical to making it work.
✕
 Remove Ads
Additionally, once the local agent is connected to the servers where you host your Docker containers (which can be multiple servers), it starts pulling data directly from Docker and presenting everything in one place. 
This includes your running and stopped containers, uptime, health states, and host-level system metrics, all of which appear without needing to jump into Portainer or open a 
terminal
 session.
 That's a huge improvement in workflow once you start using it.
Normally, checking if my stack is fully healthy means opening Portainer, scanning through the containers, or SSH-ing into the server to run Docker commands manually. If Immich stops behaving, or if Home Assistant is acting up, I have to drop back into Portainer or SSH to look into it. HomeLabInfo puts that information front and center in front of you. It's also been great for detecting old containers that I'd forgotten about and were just sitting around, running on my system. Keep in mind that HomeLabInfo offers no way to manage these containers. You'll still need to fall back on Portainer for that, but the tool works great for surfacing all the relevant information you need to work with these containers.
The project uses a hub and agent model for this. The hub runs the 
main dashboard
, while the agent connects to the machine itself and reports Docker activity, CPU usage, RAM usage, disk usage, and system health. That hub and agent model means that you're not just bound to a single server for surfacing your information; you can spin up the agent on all your machines and display them in a single spot.
✕
 Remove Ads

                        Network discovery makes HomeLabInfo way more useful
               

            A live view of devices, containers, and how your homelab connects
    
The second, and arguably bigger reason I keep it running is the 
network discovery
 that it offers. Arguably, that's where it gets more interesting. While most dashboards focus solely on appliances, HomeLabInfo places equal importance on the network itself. It scans the LAN and starts identifying devices across your subnet. That includes everything from routers to access points, NAS drives, IoT devices, and more.
Since HomeLabInfo works directly with host networking, it has all the right permissions to make this work. Once I changed the scan range to match my actual subnet, the dashboard started filling up quickly and gave me a quick glance summary of all the devices on my local network.

There's a built-in topology view too that lets you quickly visualize your entire 
home stack
. However, I found it a bit lacking, and the interactive node doesn't scale perfectly to the viewport. Unlike other tools that usually only show the topology of network devices, this view can show the topology of both devices and containers in your stack, which makes it more interesting to me, and I hope it improves over time.
✕

            HomeLabInfo is the perfect companion for your existing container manager
    
In my opinion, the best way to think about HomeLabInfo is as a dashboard or accompaniment to standard container manager tools like Portainer or your router's interface. Portainer is still there to manage your containers, and you've got your standard router interface too. HomeLabInfo, however, unifies the two interfaces and gives you an overview of your entire stack, and helps you understand exactly what's happening across your home lab in the moment. This isn't really meant to launch apps; you can use one of many other 
Docker dashboards
 for that. Instead, the goal here is to give you an overview of insights, and HomeLabInfo excels at that.
