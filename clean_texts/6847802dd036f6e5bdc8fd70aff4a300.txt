This weekend, set some time aside to focus on a few basics in your homelab. From setting up Tdarr to keep your media library nice and optimized to making Docker management easier with Portainer, and even setting up Nginx Proxy Manager for reverse proxies, here are three homelab projects for you to try out this weekend.
✕
 Remove Ads

                        Automatically transcode your library with Tdarr
               

            H.264 is so 2010
    
Credit: Patrick Campanale / How-To Geek
 For the longest time, I just ripped media to my Plex server in whatever format it defaulted to. A vast majority was H.264, while some newer content was H.265/HEVC. The problem is, H.264 is a wildly inefficient container to use in 2026. While it’s still the industry standard, H.265 is a much better option to use whenever possible—and that’s where 
Tdarr
 comes in.
Instead of having to re-rip entire TV series back to my NAS, I chose to use Tdarr to transcode the content from H.264 to H.265, and it works pretty great. There were one or two TV show episodes and movies that I did have to re-rip, but Tdarr did a great job at handling everything else for me.
✕
 Remove Ads
With Tdarr, you can tell it exactly what you want it to do. I had mine set to only transcode movies or TV shows that were above a certain bitrate, so things that were already optimized it didn’t touch. I also had it strip out extra audio channels, subtitles, and any other extra information that didn’t need to be there.
At the end of the day, 
using Tdarr in my setup saved me 7TB of storage
 without having to do much else besides set it up at the beginning.
If your Plex or Jellyfin library has grown too big for its own britches, then try out Tdarr. It might take a bit to get set up, but once it’s configured, you’ll be surprised at just how much space you can save.

                        Make managing your Docker containers easier with Portainer
               

            No more Docker CLI for you!
    
I couldn’t manage my homelab without Docker. Every system in my homelab runs Docker in some way, shape, or form. It’s on all three of my NAS systems I’m running, and exists on just about every virtual machine I use, too. I even use Docker on my VPS. It’s safe to say that I’m a Docker fanatic.
✕
 Remove Ads
While I use Docker daily, I 
hate
 Docker CLI. I simply never took the time to learn it because I started using Docker on Unraid, which handles everything through a user interface. That’s why, when I started moving Docker containers off of Unraid, 
I had to find another solution
 that kept me out of Docker CLI—and that solution is 
Portainer
.
Portainer is a simple-to-use Docker management interface. It allows you to spin up individual containers or “Stacks,” which are basically Docker Compose files.
I have Portainer running on one of my virtual machines, and then I use the Portainer Agent on all of my other Docker systems so I can control everything from one convenient UI.
Portainer is really simple to set up
, and requires just two commands to get up and going. So, if you’re still using Docker from the command line, it’s time to upgrade your setup and deploy Portainer today.

                        Set up a reverse proxy for your self-hosted services using Nginx Proxy Manager
               

            Internal or external access, it doesn’t matter—just please stop using IPs and ports
    
Credit: Nginx Proxy Manager
✕
 Remove Ads
 My homelab houses a lot of services that I use both internally and externally. Personally, 
I use Nginx Proxy Manager
 to handle all of my reverse proxy needs. I’ve used it for the past five or more years, and currently run nearly 50 reverse proxy domains through it.
Nginx Proxy Manager
 isn’t the only reverse proxy tool around. There’s Caddy, Traefik, and even 
Cloudflare Tunnels
. I choose Nginx Proxy Manager mostly because it’s what I know, and it works well.
To get Nginx Proxy Manager up and running is pretty simple. The only thing you might need to change for the Docker container is what ports it runs on, but that also is dependent on what router you have (if you plan to use it for external access).
In my setup, I have it running on ports 81 and 444, one number up from the normal 80 and 443 that HTTP and HTTPS runs on. This is because I already have other services running on 80/443 on that server. However, I’m only able to do this because I can route external port 80 to internal port 81, and external port 443 to internal port 444 with my Unifi system.
✕
 Remove Ads
When I set up Nginx Proxy Manager at my dad’s house, his router didn’t have that ability, so we had to change what ports his host OS used so that Nginx Proxy Manager could use 80/443. Regardless of your setup, you’ll need to 
port forward
 external port 80/443 to Nginx Proxy Manager if you want to access your reverse proxied services outside your local network.
After you get these basic homelab services up and running, 
you should really check out Grafana
. I was able to build a pretty sweet dashboard using Grafana that showed me quite a lot of information in my homelab at a quick glance. It took me a few hours to get Grafana set up the way I wanted though, so you should definitely have ample time set aside before starting on the project.
