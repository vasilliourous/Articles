                        Claude Code can draw...but not in the way I expected
               

            It skipped using Paint entirely
    
 So, of course, the logical next step I wanted to take with Claude Code was to ask it to draw. So I told it to open up Paint and doodle a cute kitty cat.
I wasn't sure what to expect; after all, Claude's mouse control is quite limited. It can't draw in smooth strokes, and it updates itself using intermittent screenshots, so I didn't know how it would handle this. Turns out, Claude had an amazing idea; it instead downloaded Pillow, a Python library, to do the job.
I saw the permission to install Pillow pop up, so I accepted it and asked Claude why it downloaded Pillow:
I used it because drawing in Paint by controlling the mouse pixel-by-pixel would have been really tedious and the result would've looked quite rough. Instead, I:
Installed Pillow
 (a lightweight Python library for creating images)
                        
Wrote a Python script
 that drew all the kitty's shapes mathematically — circles for the head/body, polygons for ears, arcs for the tail, etc.
                        
Saved it as a PNG
 and opened it in Paint
                        
So Paint was still the star of the show for displaying the final artwork — I just used Python as a smarter "brush" to create it!

                        Claude Code can just be your ultimate lazy companion from anywhere
               

            Give it tasks to do remotely
    
 Probably my favorite use case was the Dispatch feature. This allows you to connect your phone to your PC and send Claude commands. Claude will then use Claude Code to get the job done and report back once it's complete, or if it hits a problem. Naturally, this opens up a ton of possibilities, such as asking Claude to boot up a server while you're out, check the status of your PC, and even upload files to the cloud so you can access them on your phone.
However, you can also be really lazy. Picture the scene: you're lying in bed, and you want to play World of Warcraft. So you send a message to Claude Code via Dispatch to boot up the game and log in to a specific character. You wait a little while, then you get the message that it's all done. You hop on your PC, and sure enough, Claude has gone through the process of opening 
Battle.net
, booting World of Warcraft, waiting through the login process, then selecting your chosen character and logging in as them. Sound good? Well, that's exactly what it did for me, and I love it.

Obviously not. I don't know how someone thought these examples were worthy of writing an article.
I recently started a small business. I looked at QuickBooks for tracking expenses...I felt like they overcharge for even their cheapest subscription, because "business". Instead of just making a fancy spreadsheet, I worked with Claude Code to build myself a lightweight accounting app. I can import my transactions automatically and have an LLM categorize them for me. I have a limited mobile app that lets me create or check expenses anywhere. I can spit out reports in a clean format to give directly to my accountant, and everything I need for my quarterly tax estimates. 
After I have my accountant prepare my business taxes next year, then I may leverage that result to just automate it myself.
My brother does consulting for state agencies and nonprofits. He had been managing his client relationships in Excel. We built a proper CRM using Claude Code in a weekend. It tracks contacts easily, creates a meeting prep summary of recent communications and notes, prompts and offers follow-up on stale leads, and summarizes and details contracts. It integrates with M365 for communications, and it integrates with the accounting app I built.
I made a personalized tongue-in-cheek murder mystery table top game for my mother in law for her birthday with Claude Code. All the character profiles, interviews, pieces of evidence, etc, are created as markdown files and managed in a git repo. Entity relationships are tracked in a database and gameplay is tested and simulated in a browser app. I used Claude Code to make a custom page/type setting app because I didn't like the limited options with free ones I've used online in the past. Just took it to a print shop and had 2 copies made.
That's just the tip of the iceberg and a few days of work after I started really playing with it.
If you can't imagine how this technology will change things, then you need to think harder.
