Self-hosting tools on a home server is usually the starting point for every beginner. As you mature in the journey, you find that hosting most of the tools as 
Docker containers
. There are multiple upsides to this approach because you can isolate services from each other and manage them independently. However, it becomes an uphill task to manage everything after you set up a dozen or more services on the server.
✕
 Remove Ads
There are multiple dashboard options, like Beszel, or you can use the terminal or Portainer to check the container stats and availability. Uptime Kuma is also an option, but each of these comes with its complexities, and some can be a bit difficult to set up. I found solace in Glances, a simple tool that shows me everything useful related to my Docker setup in one go.

                        Docker monitoring should be simple
               

            Looks aren't important
    
 Ask any beginner about Docker monitoring setup, and you’ll notice a pattern. Most of the tools available are either too overwhelming or too complex. Grafana is a great example of a tool that packs every possible monitoring feature that you can imagine, but it's an uphill battle to get going. I avoid the 
Grafana and Prometheus
 combo on low-end servers that lack the resources to spare.
Uptime Kuma
 is another monitoring tool that you can use to check the container uptime, but it only does that. You can configure notifications with it, but Kuma doesn’t show anything else. You have to individually add each Docker container for monitoring, and then it’ll keep track of the uptime.
I wanted to avoid tools that showed too many graphs and was looking for something that would show me everything on one single page. Glances is a tool that offers exactly what I want without any of the added complexities.

                        Setting up Glances
               

            One run command is enough
    
Glances
 is a monitoring tool that looks like 
htop
. You must be familiar with 
htop 
because it's a system monitoring tool that you can run from any popular Linux distro terminal. Glances has a similar look and feel, but for your Docker systems. I call it “like htop” because it's one of the few tools that you use via 
SSH
, as a web dashboard, and your system's command line.
To install it on your Docker system, you only need to execute a 
Docker run
 command or create a compose file. It'll create a new container for Glances and will map the 61208 port to access its web UI. You can type the server IP:61208 to access the web dashboard. That's all.
If it were for Prometheus and Grafana, you would need to deploy a stack and then add individual resources to monitor in Grafana. Glances doesn't need any such pampering and loads up instantly without needing any further customization.

                        Everything at a glance
               

            Less is more
    
✕
 Remove Ads
Glances gets rid of the GUI approach and focuses more on telling you everything on a single page. On the top left side, you can spot the container ID assigned to Glances with the IP address, date, and time.
The left-hand side mentions system hardware details with separate sections for network, disk I/O, and file system. On the top right side, you can see more CPU, memory, and swap details. Next is the most important part, which shows Docker containers running on your system. The data is updated in real-time, so you don't need to worry about incorrect or delayed reporting.
Glances automatically sorts containers by CPU consumption, but you can sort them by memory consumption or naming scheme. The container monitoring section shows the status, uptime, CPU and memory usage, network usage, and even mentions the port number on which the tool is hosted. If you compare it with the 
Docker ps
 command
, you can see some of the elements, but you cannot see any kind of resource consumption.
While I prefer using Glances as a container monitoring solution, I don’t see any harm in using it as a CLI resource monitor. I can see details of the system processes along with all the container details. The only point of hosting it as a Docker container is that I don’t need to use SSH to access it. I can simply use the local IP and the correct port number to launch the monitoring page.
✕
 Remove Ads
Glances lacks any kind of control over the containers. So, you cannot manage your Docker containers, which means stopping, starting, or creating containers is out of the picture. But that isn’t a massive trade-off because other monitoring tools also lack this feature. Still, it would have been nice to have at least container state management (start, stop, restart) in the tool.
Having such a feature would reduce reliance on using 
Portainer
 or a terminal for such basic tasks. The memory footprint of Glances isn’t huge and uses about 80–100 MB to work. It's a trivial amount of memory to check what’s going on with your containers. You can choose to force-start the Glances' container every time you want to access the web UI, but that would add an extra step.

Glances doesn’t show the stopped or inactive Docker containers. You can configure Glances’ web UI to only show the sections you like. It’s possible, but I won’t encourage it because there are too many elements to work with.

            Everything at a glance
    
Glances is a no-nonsense tool that I’ve begun using on all my Docker systems. It’s exceptionally easy to deploy, has a very small memory footprint, and presents information in a digestible format. I don’t need to scroll through multiple tabs, graphs, or sections, and can see everything related to the host system and Docker system with it.
While I crave more container control in Glances, I understand that the tool’s focus is on simplistic, real-time monitoring, rather than more control. Give it a try, and you’ll barely miss Grafana or Uptime Kuma.
