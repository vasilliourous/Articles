I've used Windows long enough not to panic when my PC feels slow
. The Task Manager's Performance tab shows CPU usage, and it often explains what happened. However, it gets a bit trickier when CPU usage is low, and the computer remains slow. You may think it's caused by old hardware, but that's not necessarily the case.

It can be a sign to investigate the RAM and whether Windows is running out of it, especially since Windows 11 can use up to 10GB of RAM when idle on a 16GB RAM computer. So, the answer may not be the processor when your computer is slow.

                        What actually happens when your RAM fills up
               

            Your SSD becomes a very expensive, very slow substitute
    
Afam Onyimadu / MUO
 The RAM and your computer storage work a bit like a desk and a cabinet. If the desk is full, you don't suddenly stop working; you start filling up the cabinet.
Once your RAM fills up, Windows begins moving recently used data to your page file (
pagefile.sys
). This is a file on your storage drive that allows you to retrieve that data on demand in a process called a Hard Fault.
Below are hard fault numbers you should know about:
Memory type
Typical read latency
Relative to RAM
What it means in practice
DDR4 RAM
10–15 nanoseconds
Baseline
Instantaneous from the CPU's perspective
NVMe SSD (high-end)
~50–100 microseconds
~5,000× slower
Feels fast to humans, agonizing for the CPU
SATA SSD
~100 microseconds
~8,000× slower
Noticeable stutter when paging kicks in
HDD (spinning disk)
5–10 milliseconds
~600,000× slower
System effectively freezes under heavy paging
So, RAM is still about 5,000 times faster than the fastest NVMe SSDs. If your computer constantly pulls data from the storage drive, you can expect the CPU to remain idle during this process. Hence, verifying in Task Manager shows decent CPU usage without surfacing the actual process that's keeping it idle.

                        How to catch RAM pressure while it's happening
               

            There's one number in Resource Monitor that doesn't lie
    
The right tool for this problem is the Resource Monitor
. You can launch it from the Task Manager's Performance tab by clicking the ellipsis in the top-right corner and choosing Resource Monitor. Once in the process manager, expand the 
Memory
 section, and the column to look at is Hard 
Faults/sec
. It represents the number of times per second when Windows retrieves data from storage that should have been in RAM. If the values in the column are 20 or below, that is normal; but if you consistently have values above 20 while the computer stutters, that is solid confirmation that your computer is slow.
However, as you look through these figures, avoid the mistake of assuming that because free memory in the memory breakdown is low, your RAM is exhausted. The OS caches data, but that memory is released immediately and becomes available when needed.
You can make sense of the values you see using the table below:
What Resource Monitor shows
What it actually means
Action needed
Hard Faults/sec near 0, disk activity low
RAM is comfortably handling your workload
None (look elsewhere for the bottleneck)
Hard Faults/sec climbing + disk at 100%
Page file is being hammered (RAM is overwhelmed)
RAM pressure confirmed (this is your culprit)
High committed memory, Hard Faults still near 0
RAM is heavily allocated but not yet spilling to disk
Monitor it (you're close to the edge)
Free memory low, Standby memory high
Windows is caching aggressively (Standby is still usable RAM)
Normal behavior (don't misread this as a problem)

                        Why 16GB doesn't go as far as it used to
               

            Windows 11 is a different beast than it was three years ago
    
Afam Onyimadu / MUO
 Before 2024, I would still recommend 16GB of RAM for most people, but that has changed in recent years. 
Microsoft
 admits that a 16GB computer uses up to 10GB when idle, and an 8GB computer uses up to 6GB when idle. This leaves a small margin before paging to disk, and the table below gives an idea of what you may expect depending on your workload:

            Same symptom, different cause — the diagnostic instinct applies to all of them
    
RAM pressure can cause your computer to run slowly, but it's the most overlooked cause, not necessarily the only or most common trigger.

Thermal throttling is one that I have explored extensively
. It is similar to RAM pressure in the sense that people rarely consider it when a computer is slow. But there are more common culprits, like storage health degradation. You can use 
CrystalDiskInfo
 to see your drive's health status.
You may also be experiencing a post-boot storm where Windows Updates, Windows Defender, and cloud sync apps start running almost simultaneously, 10 to 15 seconds after startup.
The difference is that while a lot of the slowdowns will be visible in the Task Manager, RAM pressure requires some extra investigation to catch.
