I got tired of my smart home going offline every time I needed to perform maintenance on a server in my homelab, so I decided to fix it by building a high availability cluster. It might not have been all that simple, but the effort was worth it to have a smart home that never goes down.

                        When Home Assistant goes down, my smart home goes down
               

            A single point of failure brings nothing but frustration
    
Credit: Corbin Davenport / How-To Geek / Home Assistant
 I’ve used 
Home Assistant
 for a few years now, and I’ve always been plagued by one particular problem: a single point of failure. Originally, my homelab had just one server in it and that server handled everything, including Home Assistant.
Often, when I had to perform maintenance on that server, my wife would go to use something in our smart home only to say “The lights aren’t working” because, well, Home Assistant was down. This is a story as old as homelabbing—maintenance causing downtime.
Since I don’t have a dedicated system running Home Assistant (like a 
Home Assistant Green
 or Yellow), I’ve had to learn to work with this limitation. Typically, I try to perform maintenance during the day now, as our smart lighting sees the most use at night.
However, now that I have my 
Proxmox
 high availability cluster set up, I no longer have to worry about Home Assistant downtime.

                        Proxmox high availability made my smart home resilient
               

            You need multiple nodes, but it’s worth it
    
I recently deployed 
a high availability three-node Proxmox cluster
 in my homelab. You technically need three nodes to run a high availability cluster, but it’s possible to do it with two nodes and a third “ghost” node that just simply votes for quorum.
✕
 Remove Ads
My homelab runs on three mini PCs
 and a few NAS systems. The mini PCs really run the show though, as that’s what I use for my Proxmox nodes. I have a mixture of hardware for the mini PCs in my homelab, but you can really use any type of system for it. In fact, finding a few cheap old office PCs on Facebook Marketplace or eBay is a great starting point if you want to dabble with high availability.
Once the high availability cluster was fully deployed, I was able to enable my Home Assistant virtual machine for high availability. What happens now is the Home Assistant virtual machine migrates from the primary node it runs on to one of the other nodes in the cluster if the primary one goes down.
Then, whenever that primary node comes back online, the Home Assistant node migrates back to the original node it was on. It’s a seamless process that happens completely automatically without me having to do anything at all.

                        Now my smart home stays online—even during maintenance
               

            So long as two of my three nodes are online, so is my smart home
    
Credit: Patrick Campanale / How-To Geek
 This might seem like a lot of work for some smart lights or plugs, but it’s entirely worth it to me. I’ve moved my homelab’s location multiple times over the years, and each time my smart home would go down for hours at a time. This meant routines would stop running, lights or plugs would stop responding, and chaos would ensue. Okay, maybe not full-blown chaos, but mild chaos would definitely arise when smart devices stopped working.

Almost all of the lighting in the house
 is controlled by a smart device in one way or another, and the household is simply used to being able to say voice commands to turn on or off lights. We also use routines and automations all the time for turning off entire portions of the house with one simple voice command.
Knowing that I should have only a minute or two of downtime from here on out when it comes to doing maintenance in my homelab. There are definitely times when I need to move things around to add new hardware, pull old hardware out, or upgrade something.

Now, my routine is going to be taking the Home Assistant server down first and plugging it in somewhere else. The VM will simply migrate from the primary server to one of the backups and then back again, providing minimal downtime. Then, once everything is moved around, I can do the same thing in reverse and everything will simply go back where it should be without me having to manually move any virtual machines.

            I’m now able to make more than just Home Assistant highly available
    
While this venture started out with a goal of making Home Assistant highly available, I’m now able to make any VM I want resilient, just like Home Assistant. For example, I have three Docker virtual machines running in my homelab—one on each Proxmox node.
Now that I’ve taken the time to solve high availability for Home Assistant, it’s trivial to make the other virtual machines highly available, too. This means that my entire homelab setup can handle losing one node and my services will still stay online.
It’s definitely a good bit of work and time investment to get a high availability setup like this up and going, but it’s so worth it in the end.
