                        I picked up a cheap OLED screen for my Pi
               

            I had no real plans for what to do with it
    
Credit: Adam Davidson / How-To Geek
✕

 I recently ordered some smart home devices online and needed to bump up the total cost of my order to take advantage of a discount code and free shipping. After browsing through the available products, I ended up buying a 
small OLED display
 for my Raspberry Pi. I had no idea what I was going to use it for, but the price was just enough to get my discount and free shipping, so I figured I'd decide how to use it once it arrived.
When the screen arrived, it was easy to hook up to my Raspberry Pi. The hardest part was deciding exactly how I was going to use it. I thought about a few different options, such as using it to give visual feedback for a 
Pi-hole instance
 or for displaying 
smart home notifications
, but these were all fairly boring ideas.
Then I spotted one of my kids' 
Tamagotchis
 lying on my desk, waiting for me to change the battery. It made me think that the low-resolution display was quite similar to the screen of a Tamagotchi, and that maybe I could use the Raspberry Pi to make a Tamagotchi of my own.
I'm always trying to find ways to be more productive, and I thought that I could gamify my work life using my own Pi-based Tamagotchi. I could earn XP by completing work and ticking items off my to-do list, and have the Tamagotchi evolve each time I earned enough XP.

                        Describing the project to Claude
               

            I got working proof-of-concept code
    
The first thing I wanted to know was whether the display would work for the idea I had in mind. The screen is only 32 pixels high and 128 pixels wide, so I wanted to make sure that I could fit everything I needed onto the screen.
I had no idea how to get things to display on the screen, and although the display came with documentation and example code, I decided to 
ask Claude
 instead. I knew that creating a working Tamagotchi that could track my productivity would be complex, so I thought getting help from an AI chatbot would be the best option.

I outlined my idea: a Tamagotchi-style game running on my Raspberry Pi. The screen would display the current character, along with XP and other stats. As I completed work tasks and my to-do list, I would gain points, and with enough points, my Tamagotchi would evolve, but if I failed to complete what I should, I would lose XP, and my Tamagotchi would revert to an earlier form.
I decided that I wanted the Tamagotchi to be inspired by pop culture characters. It would start off as a Z-list celebrity and gradually evolve into A-list celebrities and then eventually, living legends. 
Home Assistant
 would provide all the key information that the Raspberry Pi needed.
✕
 Remove Ads
Claude immediately came up with code for a proof-of-concept image that I could display on the Raspberry Pi display to see if it was good enough to move on to building the rest of the project. I sent the code to the Raspberry Pi, and immediately had a prototype image of my Tamagotchi on the display. With a couple of tweaks to the font size, it looked exactly how I wanted it.

                        Working with Claude to make the Tamagotchi work
               

            A fun way to boost productivity
    
Credit: Adam Davison / How-To Geek
 Once I knew that my productivity Tamagotchi would look reasonable on the small screen, it was time to figure out how it was going to work. Claude was incredibly helpful here, too.
We had a back-and-forth discussion about different ways I could earn XP, and how this information could be accessed by the Raspberry Pi. I decided that I wanted to track three metrics that would contribute to the Tamagotchi evolving: Work, Body, and Mind. Work would track things such as how many articles I submitted; Body would track metrics such as how long I 
remained sedentary in my office chair
; and Mind would track how much time I spent on distracting apps when I should be working.
One of the biggest challenges was creating the sprite images loosely inspired by celebrity archetypes. Claude even helped here. It built a small web app that let me upload images, convert them to 32x32 pixel art, and then edit them to suit my needs. While the resolution was incredibly small, using the app, I was able to create some reasonable images for my Tamagotchi.

                        Tinkerers no longer need to be coders
               

            Personal projects are where vibe coding makes sense
    
Credit: Patrick Campanale / How-To Geek
 Once I'd created my images, Claude gave me everything I needed to get the Tamagotchi working. It told me the 
Home Assistant helpers
 I'd need to create, built the standing/sitting and focus tracking automations, and even helped me set up an 
n8n workflow
 that would update my article count every time I marked an article as completed in 
Notion
.

It then gave me the code that would get the Raspberry Pi to poll Home Assistant and pull the relevant stats, and allow the Tamagotchi to evolve as I reached my goals. It wasn't perfect on the first try, but with a few tweaks here and there, I have a working custom Tamagotchi that evolves as I reach my productivity goals. Using has made me complete tasks that I wouldn't have completed otherwise, just because I was close to unlocking the next celebrity.
Vibe coding
 rightly gets a lot of bad press when used for 
important production code
, but this is exactly the type of project where it makes sense. It allowed me to complete a project I would never have had time to complete otherwise, and if the code fails, it's not hurting anyone other than myself. It shows that you don't need to be a coder to be able to create some impressively complex tinkering projects.
