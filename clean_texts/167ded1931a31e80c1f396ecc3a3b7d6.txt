During my early Proxmox days, I stuck with GUI virtual machines and neglected LXCs for my self-hosted projects. However, I realized my mistake fairly quickly and jumped into the LXC rabbit hole with a single TurnKey-based Debian template. Now, I’ve got dozens of LXCs scattered across standalone and clustered Proxmox workstations, and each containerized app has its own utility.

                        BentoPDF
               

            The best PDF editor out there
    
Even before I started hosting my own services, I absolutely despised cloud-based PDF editors with ad-riddled UIs, forced watermarks, and long queue times. Adobe's offerings weren’t any better, either, as I had to put up with paywalled features when modifying my documents. BentoPDF and its massive arsenal of PDF-editing tools single-handedly solved these problems. I can manipulate individual pages, tweak their contents, apply watermarks, extract tables, and sign documents – all from a neat, ad-free interface.
Better yet, BentoPDF includes tons of tools for turning other formats into conventional documents and vice versa. Besides requiring updates every now and then for new features, BentoPDF doesn’t require a lot of extra maintenance either, which earns it extra brownie points in my book (and a spot on this list).
✕

                        PairDrop
               

            It’s essentially AirDrop minus the annoying Apple-only restriction
    
Apple may have a somewhat locked-down application stack, but its robust tools deserve a lot of praise. AirDrop is one such app capable of quickly transferring files between Apple devices without requiring additional permissions or external cloud access. PairDrop is essentially its FOSS equivalent, and it provides a similar cross-platform file-sharing experience; the only difference is that it works with any device capable of accessing its browser-based web UI.
PairDrop also doesn’t have restrictions on the file size, and it’s my go-to option for syncing large ISOs, bulky documents, and entire image collections (the photo kind) between different devices when I’m too lazy to set up 
Syncthing
 links or add everything to a 
Nextcloud
 instance. And its automatic device detection facility is just the icing on the cake.

                        ByteStash
               

            A neat snippet-sharing repo
    
✕
 Remove Ads
Since I work with Docker Compose files, Terraform templates, Ansible documents, and conventional config files on a regular basis, there are times when I want to archive specific code snippets. VS Code has a couple of handy extensions that help out with snippet archival, but it’s not the only code editor I use these days. And I’d rather not rely on a cloud platform such as PasteBin to store my painstakingly coded files. That’s where ByteStash comes in handy with its snippet archival provisions.
ByteStash
 houses dozens of code blocks, templates, and even entire config files in my centralized Proxmox node, and its web UI makes accessing everything a cakewalk. It also supports syntax highlighting for everything from common programming languages to app-specific config files and DevOps-centric scripts.

                        SearXNG
               

            A metasearch engine for privacy-conscious tinkerers
    
With most search engines hellbent on tracking my browsing activity, I’ve spent a long time searching for the ideal tool that could help me browse the web without compromising my privacy. Eventually, I settled on SearXNG, a metasearch engine that combines the results from dozens of conventional search engines to answer my queries. Plus, SearXNG randomizes my browser profile before sending data to Google, Bing, and other privacy-intrusive search engines, thereby providing an extra layer of anonymity to my sessions.
Besides getting rid of sponsored links, SearXNG also offers a comprehensive collection of results. I’ve currently paired it with my Ollama-powered Open WebUI server, and despite SearXNG’s straightforward setup process, it pairs exceedingly well with my LLMs.

                        Pulse
               

            It has replaced all my monitoring tools
    
If I had to pick one FOSS agent to keep an eye on my Proxmox nodes and send me alerts when things go south, I’d always choose 
Pulse
 over all its rivals. It integrates exceedingly well with PVE nodes – to the point where it can automatically detect all the LXCs and virtual machines residing on a Proxmox system once I pair it with Pulse. It can also read the resource consumption metrics for my Proxmox server and its virtual guests, and supports remote monitoring for the rest of the PVE rigs in my home lab.

UI-wise, it’s probably the best monitoring tool I’ve used so far, and I can even fine-tune the alert settings to receive notifications for every minor event, be it an LXC closing unexpectedly or a VM siphoning more resources than it should. Heck, it’s also compatible with Proxmox Backup Server workstations and conventional Docker machines, so I can monitor practically every node in my tinkering environment from a single web UI.

                        PVE Helper Scripts Local
               

            It’s a must-have for PVE Helper Scripts users
    
✕
 Remove Ads
Considering that the Proxmox VE-Helper Scripts repository offers simple commands for everything from VM and LXC deployment to host modifications and virtual guest updates, it’s no surprise that it’s well-loved by home labbers. But the real star of the show is the 
local instance
 of this repo, which provides even more QoL benefits than its online-only counterpart.
For starters, it can execute scripts remotely on my Proxmox workstations via SSH, so I don’t have to copy and paste commands manually in the Shell tab of my PVE nodes. It can also automatically detect virtual guests deployed using Helper Scripts, and it lets me change the FUSE options, mount settings, and other obscure elements that aren’t typically accessible on the Proxmox interface. And since it supports live output, I don’t have to worry about accidentally clicking on a random tab and losing access to the session while a script is in the middle of execution.
