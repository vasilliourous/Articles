Experimenting with Docker is fun because you can run several containerized services without messing anything up. That’s how I have a bunch of containers running in my humble home lab. However, I often fail to perform maintenance tasks, such as updating containers to their latest images.

                        Why I prefer Tugtainer over Watchtower
               

            Manual control keeps a sanity check
​
Despite past problems with auto-updates, I tend to keep auto-updates enabled on every device or machine I use. Even on others’ devices. Continuing the same with Watchtower seemed convenient, except when it broke my containerized services setup a couple of times.
Typically, Watchtower polls the registry for updates, then pulls the latest image. Then it stops the existing containers and recreates them using the latest updates. Since the 
entire process is automated
, it breaks some containers more than once.
Apart from restarting at odd times, there’s no mechanism to roll back changes. It’s fine for non-essential containers, but it has messed up my self-hosted dashboards like 
Homarr
, 
Heimdall
, and 
Homepage
.
✕
 Remove Ads

                        Tugtainer is a game-changer for my workflow
               

            No complexities involved
    
Tugtainer is also a Docker updating container with a very notable twist. It fetches the updates but doesn’t automatically update any containers.
With Tugtainer, when updates are available, Tugtainer sends notifications to help me make an informed call on whether to update container(s) or skip them. Besides, the “update available” label also appears on the app’s dashboard.
It’s easy to check and approve updates through Tugtainer’s web interface. By giving me control, I can choose a safer upgrade path for my essential Dockers. Besides, it supports multiple hosts, which allows me to manage updates for Docker running on different machines in my home.

                        Setting up Tugtainer takes barely a few minutes
               

            Straightforward deployment
    
While I made the best of the label-based configuration in Watchtower, I don’t miss it. Deploying Tugtainer barely takes a minute. The app automatically discovers the existing Docker containers running on the same machine.
I love that I get to choose which containers I want Tugtainer to monitor for manual updates, in addition to auto-updates, and to exclude the experimental ones.
✕
 Remove Ads
I can set to receive notifications through email, webhook, or both ways to not miss out on any updates. On feeding the relevant credentials, the Tugtainer sends a test notification.
Tugtainer allows adding multiple new host machines and controlling their Docker updates from a single web-based dashboard. So checking all hosts at once becomes convenient, rather than fetching updates individually.
​

                        Where the difference becomes apparent and shines
               

            Choice to not auto-update everything
    
Tugtainer offers a different set of options instead of the typical 
'set it and forget it' containers
. That’s what sets it apart: I can clearly choose which containers I want it to monitor and fetch their updates accordingly.
The selective control of which containers I want to check for updates, but decide to apply them later, is what I missed in Watchtower. That way, I prevent my 
media-specific Dockers
 from automatically updating and crashing.​
Tugtainer doesn’t show me any changelogs or version numbers. I still need to check the respective container’s Docker Hub or GitHub page for that. So I get notified when the updates exist and get to choose when or whether to apply them. That control is exactly what I needed to not break things with auto-updates.
What I loved is that Tugtainer smartly picks up the linked containers (media and network) and updates them in the correct order.

            Automatic updates aren’t always better in a home lab
    
Tugtainer provides me with the control and visibility to monitor Docker container updates without manual checks. If auto-updates are breaking your containerized system in your home lab, Tugtainer is the perfect alternative to Watchtower. It’s recommended for smaller, more tightly packed setups with a bunch of self-hosted apps running in containers.
Watchtower works best
 as an update automation that fetches and applies all the latest updates. But some changes can potentially break things and render the container unavailable. Also, Tugtainer's agent system makes it easy to manage updates for containers running across different machines.
That said, you can still use Watchtower on a host with a handful of critical containers for your workflow. Meanwhile, Tugtainer can handle the lower-priority ones.

    						Tugtainer
			
A web-based app for automated and manual Docker container updates that also offers multiple hosts support.
