Self-hosting starts easily with a few services. You pick the most essential ones, like 
Docker
, a media streaming server, a local cloud storage solution, a DNS resolver like Pi-hole, and a few more. But you don't stop at that. Soon, you will have a myriad of services to manage backups, monitor services, check uptime, and more. I'm not against deploying multiple services, but accessing them becomes a pain after some time. Since each service or tool has a custom URL that you use to access the web UI, remembering each one becomes a problem.
✕
 Remove Ads
Initially, I thought that bookmarking everything in my browser in a separate folder would suffice. But that means cycling through a long list of tiny labels to access the web interface. Glance solved my problem by building a custom dashboard that shows all my services with direct links to them and a brief overview of their status as well. I no longer need to use the bookmark trick or keep multiple tabs open and waste system resources.

                        IP addresses are tough to remember
               

            It gets annoying after a point
    
 The biggest pain point of self-hosting is remembering the correct port number of each service. Bookmarking is a temporary relief that becomes confusing as the list grows. My previous workflow was keeping multiple tabs in a tab group. Whenever I launched the group, all tabs would simultaneously launch, consuming more system resources than I needed.
The second problem is the overall appearance of such an orthodox design. You need to cycle between multiple tabs and don't have a bird's-eye view of everything on one screen.
I needed a 
customizable dashboard
 interface that was lightweight, supported monitoring a variety of services, and could include some elements from my other workflow as well. Glance became a perfect match because it checked all the boxes and was deployable in a jiffy.

                        Setting up Glance
               

            Only need to deploy one container
    
There are a variety of methods to deploy Glance. Don't confuse it with “Glances” because that's another tool for monitoring, while the one I'm using is a dashboard. Simply grab the 
default compose file for Glance
 and create a new folder to save its assets. You can also use 
Docker Run
 for it, and no matter what mode you choose, the container will be up and running in no time.
However, that's only half the story. Glance can display something only after you create and edit the YML file. Before complicating things, I went with a basic YML file that would create a few columns on a single page. It shows a calendar, an RSS feed, and some more things. It loads instantly and fetches the info from the mentioned sources.

                        Customizing Glance
               

            Putting all services on the front
    
 Glance doesn't automatically show all the self-hosted services that you use with a label and a URL link. You must do the heavy lifting to add each service to the dashboard page. I tried adopting a shortcut by adding a Docker widget to Glance. To achieve it, I specified the Docker volume bind in the Glance compose file. After reloading, Glance displayed all the available Docker containers with their correct uptime and status.
✕
 Remove Ads
Problem solved, right? Well, it works for the most part, but it has a tiny problem. I can see the Docker container’s name, uptime, and availability. But I cannot open its web UI by clicking on the label. Glance cannot individually extract the relevant URLs and then turn each container tile into a clickable link.
I had two options now. I could create individual widgets for each 
Docker container
, specify the correct URL, and then turn it into a clickable link. Or I could use the services template and add a label, an icon, a URL, and other details. I went with the latter and added all my containers and other services as well. It took some time to do so, but the effort was satisfying in the end.
Glance has a rich configuration system to add and specify more details about a service or container. You can use a small or full column design, and it'll automatically place the elements accordingly.
 I use a single-page design, but if you have multiple services and want more elements like feeds, Reddit, and more, a multiple-page design would be the most beneficial. My convoluted setup is now a single-page design that shows every update I need without opening new tabs. The feeds tell me about the latest stuff, and the services show the current status and provide a gateway to directly open the page.

There are several theming options available for 
Glance
 to change the color scheme and a few more aspects. I love the default layout and didn't bother with Glance UI customization. My focus was more towards making everything accessible on the front page. I also set up the Glance page as my default loading page whenever I open the browser. So, everything is available immediately after I launch the page.

            From chaos to orderliness
    
It feels less chaotic after deploying the Glance dashboard because it shows my self-hosted tools on one page. I no longer open twelve tabs in one go and get to view additional information related to my feeds, YouTube channels, weather, and more. All of this happens with just a one-time configuration of the YML file and doesn’t need any additional elements.
You can also add elements like server stats monitoring using a Glance agent or add your favorite Twitch streamers or Docker containers with custom templates. If the service is offline, you'll see a warning sign next to it.
