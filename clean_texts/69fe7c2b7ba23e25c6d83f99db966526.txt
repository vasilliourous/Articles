Linux 7.1 has just entered the release candidate phase, and if it proves to be stable enough in the coming weeks, we should see it released fairly soon. The new version removes support for the i486 CPU, which was released in 1989, because it was infeasible to continue maintaining it when nobody was likely still using it.
✕
 Remove Ads
While some people lamented that Linux was dropping support for a piece of hardware, the truth is, I feel Linus needed to do so. And not only that, but I think the kernel would be better off cutting even more.

                        The idea of "Linux runs on anything" can be detrimental to the kernel
               

            Functionality should come before pride
I think a big reason people were sad that Linux was losing support for the i486 processor was due to the biggest point of pride that they have with the kernel: the idea that Linux can run on everything. The idea that Linux would no longer run on something specific was obviously a let-down. After all, who doesn't want the pride of knowing that Linux could, in theory, run on a processor that's almost 40 years old?
However, there's a very real and human cost to ensuring that Linux runs on literally everything. All updates have to ensure they play nicely with a piece of hardware that nobody is realistically running Linux on. And every consideration for a computer so old that people probably aren't actually running it anymore drains energy that would be best going elsewhere.

                        The people who are running the hardware probably don't care anyway
               

            I can't imagine someone running an i486 downloading new Linux kernels
    
✕
 Remove Ads
 Let's assume for a moment that someone does, in fact, have Linux running off an i486. If that's the case, my next question is: are they really itching to download the latest Linux kernel? Are they looking at Linux 7.0 and rubbing their hands with glee as they imagine running the absolute bare minimum on their near-40-year-old CPU? Probably not.
In fact, I'd wager that they found an old version of Linux, threw it on their PC, and left it at that. Sure, the version of Linux they're using is no longer supported and vulnerable to attack, but I don't think an i486 PC is going on the internet. Apparently, there are some hacks and tricks you can do to actually get an i486 on the internet, but you end up with a limited and miserable experience, to the point where I really don't think a hacker is going to sneak onto your system. I don't think there'd be enough room on your hardware to even install a backdoor.

                        However, maintaining some legacy support is good
               

            I don't want all of it to go away
    
So am I advocating that Linus simply does away with any hardware that was released before 2022 and focus on the newer stuff? Absolutely not. As much as I think the idea of "Linux can run on anything" can harm the kernel, I think, when done correctly, it can be a really cool aspect of the system. It's just that we have to be reasonable with it.
✕
 Remove Ads
For example, there are some really old PCs out there that have internet connectivity, are very old, and need an OS that can offer them protection. For them, Linux is a perfect fit. And if Linus ever decides that those people no longer deserve Linux updates, then I'll be pretty upset with the decision.
However, some use cases really should have been axed sooner. Stuff like the i486 processor, and the recent Linux 7.0 update that dropped support for a specific kind of cable only used in a university a few decades ago, probably should have been removed a lot earlier. By cutting the chaff, the maintainers can maintain focus on the areas where their attention truly matters without hindering anyone

            Legacy support needs a reason to stay
    
Supporting legacy hardware just for the sake of supporting it sounds great in practice; in reality, it can be a real drain on resources for zero return. While Linux does, and should continue to, run on older hardware, some fringe cases should be allowed to pass on as a memory.
